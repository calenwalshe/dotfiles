## Idea (verbatim)

I have a Google mesh network at home with Google speakers scattered around the house — some small ones on the mesh points and some bigger higher-quality Google speakers. I have a vinyl turntable that goes through a phono amp to speakers with an A/B selector for two zones. I also plan to add a CD player and start collecting CDs. I want to: (1) control music playback on all Google Cast speakers via natural language from Claude Code, (2) distribute vinyl and CD audio from physical media to the whole house through the Google speakers, (3) play from Spotify, YouTube Music, and YouTube via Cast, (4) play my own music collection through Cast. I run Claude Code on a remote VPS and have a Raspberry Pi at home that hasn't been set up yet.

## Goal

A unified whole-house audio system where vinyl, CD, personal music collection, Spotify, YouTube Music, and YouTube video audio are all controllable via natural language commands from Claude Code, playing through any combination of Google Cast speakers throughout the house.

## Non-Goals

- Replacing the existing turntable/amp/A/B speaker setup (it stays as-is, we ADD Cast distribution on top)
- Building a custom music streaming service or recommendation engine
- Supporting non-Google cast targets (AirPlay, Bluetooth point-to-point, Sonos native)
- Video output to TVs (audio focus, though YouTube audio-from-video is in scope)
- Multi-user profiles or per-person preferences
- Home automation beyond audio (lights, thermostats, etc.)

## Roles

- **User is the hardware implementer and designer** — will physically set up all hardware, run cables, configure the Pi, place devices. Claude builds the software and provides the hardware shopping list + wiring guide.

## Constraints

- **Hardware budget: $200 max** (covers all new purchases — cables, adapters, audio interface, SD card, case, etc. Pi itself already owned.)
- Must work from Claude Code CLI on remote VPS (144.202.81.218, Vultr)
- Google Cast protocol is LAN-only — requires a bridge device on the home network
- Raspberry Pi is available but not yet configured — it becomes the home agent
- Vinyl capture requires hardware: USB audio interface connected to phono amp output (new purchase)
- Cast devices use Google DNS (8.8.8.8) — media URLs must be publicly routable OR served from the LAN
- Spotify and YouTube Music have DRM — cannot stream arbitrary tracks via raw HTTP; must use their Cast-native integrations (Spotify Connect, YouTube Music cast receiver)
- YouTube video audio must go through yt-dlp or Cast-native YouTube receiver

## Assumptions

- User has a Google Mesh network (Nest Wifi) with multiple Google Cast speakers:
  - Several smaller speakers (likely Nest Mini / Home Mini) on mesh points
  - At least one larger speaker (likely Google Home Max or Nest Audio) — better quality
- Turntable → phono amp → A/B speaker selector → 2 physical speaker zones (analog, working)
- Phono amp has an available output (tape out, pre-out, or headphone jack) that can feed a USB audio interface without disrupting the existing speaker setup
- Raspberry Pi is a Pi 3B+ or Pi 4 (has USB ports, WiFi, enough CPU for audio capture + streaming)
- User has Spotify Premium (required for Spotify Connect / Cast integration)
- YouTube Music and YouTube are accessible via Google account already linked to Cast devices
- Home network provides DHCP and the Pi can join the Google Mesh WiFi or connect via ethernet
- Music collection exists but location/format not yet confirmed — may be on a home machine, NAS, or needs to be transferred

## Open Questions

1. **Raspberry Pi model?** (Pi 3B+, Pi 4, Pi 5, Pi Zero?) — determines USB/CPU capabilities
2. **Phono amp outputs?** Does the amp have a tape-out, pre-out, or second output that can feed the Pi without cutting off the existing speakers? Or do we need a Y-splitter?
3. **Music collection location and format?** Files on a home computer, external drive, NAS, or not yet consolidated? What formats (MP3, FLAC, etc.)?
4. **Which Google speakers exactly?** Running `catt scan` from the Pi will answer this definitively, but rough count helps for planning Cast groups
5. **Spotify account tier?** Premium is required for Spotify Connect / programmatic Cast control
6. **Acceptable latency for vinyl distribution?** Cast group playback has ~1-2 second buffer delay — is that OK for vinyl listening?
7. **YouTube use case** — is this "play this YouTube video's audio on the kitchen speaker" or "play YouTube Music playlists across the house" or both?
