## Summary

All confirmed equipment is compatible. One signal level nuance needs attention: the Fluance RT80's LINE output is 140mV (low end of line-level), which may require software gain boost on the Pi capture side. The Y-splitter approach is validated for this exact use case. The Sony STR-DH190's lack of tape/pre-out is confirmed — we correctly route around it by splitting before the receiver.

## Validated Equipment Chain

```
Fluance RT80 (LINE mode, 140mV) → Y-Splitter → Sony STR-DH190 LINE IN + Behringer UCA202 → Pi
```

## Finding 1: Fluance RT80 LINE Output — Validated with Caveat

**Specs confirmed from Fluance official docs:**
- LINE output: **140 mV (-17 dBV)** via built-in Texas Instruments preamp
- Phono bypass: "All electronics are bypassed when the internal preamp is disabled"
- Connector: Gold-plated RCA
- Ground terminal: Yes (not needed in LINE mode)
- PHONO/LINE switch on back panel

**140mV is valid line-level but on the low side.** Standard consumer line-level is typically 300mV–2V (-10dBV to +4dBu). The RT80's output is about 6dB below typical consumer line-level.

**Impact on our setup:**
- The Sony STR-DH190 will handle 140mV fine — receivers are designed for a wide range of input levels, and you control volume with the knob.
- The UCA202 will capture the signal, but it may appear quiet in the recording. **Solution: boost capture gain in ALSA mixer or add +10-15dB gain in the ffmpeg pipeline before encoding to Icecast.** This is a software adjustment, not a hardware problem.

**One Pi forum user reported exactly this**: "I run into a problem where the captured sound volume is too low" when using UCA202 with a turntable. The fix was adjusting ALSA capture volume: `amixer -c 1 set 'PCM' capture 100%` or applying gain in ffmpeg: `ffmpeg -f alsa -i hw:1,0 -af volume=3.0 ...`

**Verdict: No hardware change needed.** Software gain adjustment will handle it.

## Finding 2: Sony STR-DH190 — No Tape Out Confirmed

**Confirmed via Best Buy Q&A (official Sony response):**
> "The current line of Sony receivers does not have a tape monitor loop. For this reason, it is not possible to connect an add-on equalizer to current Sony AV receivers."

The STR-DH190 has:
- 4x RCA stereo inputs (PHONO, CD, LINE 1, LINE 2/BT)
- Speaker A/B terminals (amplified output — NOT line-level, cannot feed UCA202)
- 1/4" headphone jack (volume-dependent, usable as emergency fallback but not ideal)
- Bluetooth receive only

**Our approach (split before the receiver) is the correct and only clean option.** This was already the plan — research confirms it's the right call.

## Finding 3: Y-Splitter After RT80 Built-In Preamp — Safe

**Critical distinction from audio forums:**
- **NEVER split before a phono preamp** — the cartridge's impedance loading is critical to frequency response. A Y-splitter changes the load impedance and detunes the cartridge/preamp circuit.
- **SAFE to split after a phono preamp (LINE mode)** — the preamp's output impedance is low (typically <1kΩ), and both destinations (receiver + UCA202) have high input impedance (>10kΩ). No audible degradation.

**Impedance math for our setup:**
- RT80 built-in preamp output impedance: not specified, but TI preamp chips typically <600Ω
- Sony STR-DH190 LINE IN impedance: ~47kΩ (standard)
- Behringer UCA202 input impedance: ~10kΩ (per specs)
- Parallel load: ~8.3kΩ
- Load ratio: 8.3kΩ / 600Ω = ~14:1 — well above the 10:1 minimum for voltage bridging

**Verdict: Y-splitter works perfectly here.** No quality loss, no impedance issues.

## Finding 4: RT80 Manual Confirms Clean LINE Mode

From the official RT80/RT81/RT81P user manual (Fluance):
> "If you wish to use an external preamp you can disable the internal preamp by moving the switch to the PHONO position. All electronics are bypassed when the internal preamp is disabled."

And conversely, in LINE mode, the built-in TI preamp is active and provides RIAA-equalized, amplified line-level output. This is exactly what we want — a clean line-level signal that the UCA202 can capture directly.

**Note:** The ground wire from the RT80 is not needed when using LINE mode (ground is for phono-level signals to prevent hum). However, if you experience any hum through the UCA202, connecting the ground wire to the UCA202's chassis (via a clip or tape) can help.

## Finding 5: Potential Volume Issue — Known, Solvable

The Raspberry Pi Forums thread about UCA202 low capture volume is directly relevant. The fix chain:

1. **First try**: ALSA capture volume at 100%: `amixer -c 1 set 'Capture' 100%`
2. **If still too quiet**: Add gain in ffmpeg: `-af volume=3.0` (roughly +10dB)
3. **If still too quiet**: The UCA202 has no physical gain knob, but software gain up to +20dB is clean at 16-bit/48kHz
4. **Nuclear option**: Replace UCA202 with HiFiBerry DAC2 ADC Pro which has adjustable hardware gain (-10dB to +40dB)

This is a software configuration issue, not an architecture problem.
