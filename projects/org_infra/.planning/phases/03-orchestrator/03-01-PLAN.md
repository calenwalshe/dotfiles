```python
class ResearchOrchestrator:
    def evaluate_gate(self, phase: str, artifacts: dict[str, Any]) -> OrchestratorArtifact:
        """Evaluate whether artifacts are sufficient to pass the phase gate.
        
        Rules per phase:
        - discovery: requires 'uxr' artifact with ≥1 persona with data_sources
        - definition: requires 'pm' artifact with ≥1 requirement, 'ds' artifact with feasibility
        - pitch_evaluation: requires 'evaluation', 'pressure_test' (≥1 objection), 'feedback_synthesis' (≥1 alignment + ≥1 conflict)
        
        Returns OrchestratorArtifact with cited_artifacts listing which artifacts were checked,
        and gaps listing specific missing/incomplete items on reject.
        """
    
    def update_understanding(self, phase: str, artifacts: dict[str, Any]) -> str:
        """Generate current best understanding document from all available artifacts."""
```

Key design:
- Gate evaluation is deterministic (rule-based), not LLM-based
- Every approve decision must list which artifacts were checked
- Every reject decision must list specific gaps (e.g., "UXR artifact has 0 personas")
- Understanding doc is a structured text summary of what's known so far
