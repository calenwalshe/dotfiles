  <name>Task 1: Write agent role contracts document</name>
  <files>docs/agent-contracts.md</files>
  <action>
Define contracts for all 7 agents. For each agent, specify:

**1. Research Orchestrator**
- Input: Product problem statement (NL text), all agent artifacts from current phase
- Output: Gate decision (approve/reject with cited artifact references), "current best understanding" document
- Tool access: Artifact store (read all), LangGraph state (read/write)
- Evaluation rubric: Gate decisions cite specific artifacts; rejection includes specific gaps

**2. UX Research Agent**
- Input: Problem statement, available user research data
- Output: UXRArtifact (personas, problem validation, user signal synthesis)
- Tool access: Meta internal research repos (read), artifact store (write)
- Evaluation rubric: Personas grounded in data (not invented); problem validation cites evidence

**3. PM Agent**
- Input: Problem statement, UXR findings, prior phase artifacts
- Output: PMArtifact (product pitch, requirements, prioritization)
- Tool access: Artifact store (read/write), comms interface (send/receive)
- Evaluation rubric: Requirements are testable; prioritization has explicit rationale

**4. Data Science Agent**
- Input: Problem statement, UXR findings, PM requirements
- Output: DSArtifact (feasibility assessment, data availability, experiment design)
- Tool access: Meta data systems (read), artifact store (write)
- Evaluation rubric: Feasibility claims reference specific data sources; experiment design is actionable

**5. Evaluation Agent**
- Input: All prior phase artifacts
- Output: EvaluationArtifact (success criteria, test harness concept, eval schema)
- Tool access: Artifact store (read/write)
- Evaluation rubric: Success criteria are measurable; test harness concept maps to requirements

**6. Pressure Testing Agent**
- Input: Product pitch, requirements, UXR findings, DS feasibility
- Output: PressureTestArtifact (adversarial challenge report with named objections)
- Tool access: Artifact store (read), all prior artifacts (read)
- Evaluation rubric: Objections are specific and named (not generic); each challenges a concrete claim

**7. Feedback Synthesis Agent**
- Input: Stakeholder responses from comms interface, all internal artifacts
- Output: FeedbackSynthesisArtifact (alignment/conflict report)
- Tool access: Comms interface (read), artifact store (read/write)
- Evaluation rubric: Surfaces at least 1 alignment and 1 conflict; conflicts reference specific internal findings

Use markdown table format for each agent's contract summary, then detailed sections below.
