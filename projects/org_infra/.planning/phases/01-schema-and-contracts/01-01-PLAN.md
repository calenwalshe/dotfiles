  <files>src/schemas/handoff_package.py</files>
  <action>
Define Pydantic v2 BaseModel classes for the handoff package. Top-level model `HandoffPackage` with typed sub-models for each of the 9 sections defined in the spec:

1. `problem_statement` — validated problem statement text + supporting evidence
2. `user_research` — personas, problem validation findings, user signal synthesis
3. `product_pitch` — product brief / pitch document
4. `requirements` — prioritized requirements list (what to build, what not to)
5. `eval_criteria` — measurable success criteria
6. `test_harness_concept` — structure and intent of tests (not implementation)
7. `feedback_synthesis` — stakeholder feedback report (alignments + conflicts)
8. `risk_log` — identified risks with mitigations
9. `open_assumptions` — assumptions for engineering to resolve

Each sub-model should have typed fields (not just `str`). Use `list[str]`, `dict`, and nested models where appropriate. Include `metadata` field on `HandoffPackage` with `schema_version`, `created_at`, `run_id`, `source_agents`.

Use `from __future__ import annotations` and Pydantic v2 patterns (model_config, field_validator where needed).
  </action>
