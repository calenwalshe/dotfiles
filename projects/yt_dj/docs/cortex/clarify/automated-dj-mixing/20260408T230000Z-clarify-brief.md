## Goal

An automated DJ mixing system that takes the curated track library (music/clips/) and produces continuous, beat-matched, crossfaded mixes suitable for the YouTube webcam live stream — running autonomously without manual track selection or transition timing.

---

## Non-Goals

- Not building a DJ performance tool for live human mixing (Serato/Rekordbox replacement)
- Not doing real-time beat detection or key analysis from scratch — use existing libraries
- Not building a music recommendation engine — tracks are pre-curated by the owner
- Not handling music production, mastering, or effects beyond mixing transitions
- Not replacing the owner's ability to manually intervene via chatbot — this is the autonomous fallback

---

## Constraints

- Must work with the existing track library in music/clips/ (MP3 files, 3-10 min each)
- Must integrate with the existing pipeline: output audio that Liquidsoap or FFmpeg can consume
- Must run headless on Linux (no GUI, no X11 dependency beyond what OBS already uses)
- Must produce smooth transitions — beat-matched crossfades, not hard cuts
- Must be controllable via the Redis command bus (genre/mood changes, skip, energy adjustments)
- Must handle tracks of varying BPM and key without jarring transitions
- Should be able to run 24/7 without running out of tracks (loop/shuffle the library)

---

## Assumptions

- Existing Python audio analysis libraries (librosa, essentia, madmom) can reliably detect BPM and musical key from the track library
- Beat-matched crossfading is achievable programmatically without a full DJ engine
- Liquidsoap may already handle most of this (crossfading, playlist scheduling, metadata) and just needs configuration
- The track library is large enough (~40+ tracks) to avoid obvious repetition over a few hours

---

## Open Questions

- Can Liquidsoap handle beat-matched crossfading natively, or does it only do time-based crossfades?
- What open-source tools exist for automated DJ mixing with beat matching? (e.g., Mixxx headless, AutoDJ engines, python-audio-separator, demucs for stem separation)
- Should we pre-analyze all tracks (BPM, key, energy) and store metadata, or analyze on-the-fly?
- What's the right transition strategy: harmonic mixing (key-matched), BPM-matched, energy-curve based, or all three?
- How to handle large BPM jumps between genres (e.g., 90 BPM downtempo → 128 BPM house)?
- Should the system generate full pre-mixed sets, or mix in real-time during the stream?
- Is there a headless mode for Mixxx that could be controlled programmatically?

## Next Research Steps

1. Evaluate Liquidsoap's crossfading capabilities — smart crossfade, beat-aware transitions, what it can and can't do
2. Survey automated DJ mixing tools — Mixxx headless, AutoDJ engines, python libraries (pydub, librosa, essentia)
3. Research beat detection and key analysis libraries — accuracy, speed, Python integration
4. Investigate harmonic mixing algorithms — Camelot wheel, key compatibility, open-source implementations
5. Look at existing open-source auto-DJ projects on GitHub
6. Assess whether pre-analysis (BPM/key/energy database) or real-time analysis is more practical
7. Evaluate transition strategies for mixed-genre libraries with varying BPM ranges
