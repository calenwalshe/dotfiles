## Goal

A self-hosted internet radio station running on the agent-stack box that streams the owner's MP3
library via Icecast2 + Liquidsoap, paired with a web frontend that embeds curated public webcam
feeds client-side — expressing the owner's personal aesthetic, accessible to a small private
audience with no YouTube dependency.

---

## Non-Goals

- YouTube Live streaming (this is the pivot away from it)
- Large audience scaling or CDN
- Video encoding / RTMP / FFmpeg video pipeline
- OBS, Xvfb, frame_mixer, solar_scorer (all dropped)
- 12-hour auto-restart logic (YouTube-specific problem)
- Content ID or copyright mitigation
- Chatbot-driven runtime commands (v1 — may revisit)
- Music licensing compliance (owner's explicit informed choice)
- Mobile app or native client
- Multi-platform simultaneous streaming

---

## Constraints

- Runs on the agent-stack box via Docker Compose (existing operational pattern)
- Commercial internet connection — bandwidth not a ceiling
- Audience is small and private ("pirate radio") — no CDN, no horizontal scaling
- Audio source is owner's personal MP3 library only
- No RTMP output — pure HTTP audio streaming (Icecast protocol)
- Webcam feeds must be embedded client-side — server does not transcode or proxy video
- Must coexist with existing agent-stack services without port conflicts

---

## Assumptions

- Agent-stack box has Docker and Docker Compose installed (per CLAUDE.md operational pattern)
- Icecast2 and Liquidsoap can each run as Docker Compose services
- Windy Webcams and DOT feeds are publicly embeddable without CORS issues for client-side use
- Existing `config/liquidsoap.liq` is reusable or minimally adapted
- Existing `config/cameras.json` (30+ feeds) is reusable as the webcam selection pool
- Nginx can serve as reverse proxy for Icecast and the web frontend from a single port
- MP3 library is accessible on the agent-stack box filesystem (mounted volume)
