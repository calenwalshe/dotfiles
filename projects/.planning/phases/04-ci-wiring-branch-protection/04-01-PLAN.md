# Plan 04-01: GitHub Actions Workflow and Branch Protection

## Objective

Write `.github/workflows/ci-evals.yml` that runs the eval suite on PRs and fails when `eval-gate.py` exits 1. Configure branch protection on `main` to require the `ci-evals` status check. A failing CI job does NOT block merge without branch protection — both are required.

## Execution Context

- Project root: `/home/agent/projects/`
- Branch: `cortex/experiment-control-plane`
- Job name MUST be exactly `ci-evals` — this is what branch protection will reference
- Phoenix URL: `http://144.202.81.218:6006` (public IP for GitHub Actions runners to reach)
- UFW port 6006 already open to 0.0.0.0/0 (Phase 1)
- PHOENIX_URL should be set as a GitHub Actions variable (not secret, not sensitive)
- OPENAI_API_KEY should be set as a GitHub Actions secret (sensitive)
- Branch protection requires GitHub API access or manual configuration in Settings UI

## Tasks

### Task 1: Create workflow directory and write ci-evals.yml

Write `/home/agent/projects/.github/workflows/ci-evals.yml`:

```yaml
name: CI Evals

on:
  pull_request:
    branches: [main]

jobs:
  ci-evals:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-python@v5
        with:
          python-version: "3.11"
          cache: "pip"

      - name: Install eval dependencies
        run: pip install "arize-phoenix-evals>=2" pandas pytest

      - name: Run eval suite
        env:
          OPENAI_API_KEY: ${{ secrets.OPENAI_API_KEY }}
          PHOENIX_URL: ${{ vars.PHOENIX_URL }}
        run: pytest tests/evals/test_ci_gate.py -v

      - name: Check eval gate
        run: python scripts/eval-gate.py

      - name: Upload eval results
        if: always()
        uses: actions/upload-artifact@v4
        with:
          name: eval-results
          path: evals/current.json
          if-no-files-found: ignore
```

Create the directory first:
```bash
mkdir -p /home/agent/projects/.github/workflows
```

### Task 2: Configure GitHub Actions secrets and variables

Check if the GitHub CLI is available and if the repo has a remote:
```bash

```

If `gh` is authenticated and repo has a remote:

Set OPENAI_API_KEY secret:
```bash
# Read from environment
OPENAI_KEY="${OPENAI_API_KEY:-}"
if [[ -n "$OPENAI_KEY" ]]; then
    REPO=$(git -C /home/agent/projects remote get-url origin | sed 's|.*github.com[:/]||;s|\.git$||')
```

### Task 3: Configure branch protection via GitHub API

Branch protection MUST be configured for the CI gate to actually block merges. A red CI check alone does not block merge.

Check gh availability:
```bash
gh auth status 2>/dev/null
```

If `gh` is authenticated:
```bash
REPO=$(git -C /home/agent/projects remote get-url origin | sed 's|.*github.com[:/]||;s|\.git$||')
OWNER=$(echo "$REPO" | cut -d'/' -f1)
REPO_NAME=$(echo "$REPO" | cut -d'/' -f2)

# Configure branch protection on main requiring ci-evals status check
gh api \
  --method PUT \
  -H "Accept: application/vnd.github+json" \
  "/repos/${OWNER}/${REPO_NAME}/branches/main/protection" \
  --input - <<'EOF'
{
  "required_status_checks": {
    "strict": false,
    "contexts": ["ci-evals"]
  },
  "enforce_admins": false,
  "required_pull_request_reviews": null,
  "restrictions": null
}
EOF
echo "Branch protection configured"
```

If `gh` is not authenticated, document the manual steps clearly in the summary:
```
Manual branch protection steps:
1. Go to: https://github.com/{owner}/{repo}/settings/branches
2. Click "Add branch protection rule"
3. Branch name pattern: main
4. Check "Require status checks to pass before merging"
5. Search for and select: ci-evals
6. Click "Save changes"
```
