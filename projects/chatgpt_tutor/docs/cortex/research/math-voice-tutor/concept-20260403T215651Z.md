# Research Dossier — Concept Phase

- **Slug:** math-voice-tutor
- **Phase:** concept
- **Timestamp:** 20260403T215651Z
- **Depth:** standard

## Summary

The Math Voice Tutor spec is architecturally sound for its platform constraints. ChatGPT Projects with project-only memory, voice mode with image upload on iOS, and structured project instructions form a viable stack for guided tutoring sessions. However, research reveals three material risks: (1) ChatGPT's math step-by-step accuracy is poor (~42% incorrect solution approaches), (2) Study Mode is not grade-calibrated and sometimes gives direct answers, and (3) OpenAI's terms exclude children under 13 from the service entirely.

## Findings

### 1. ChatGPT Projects and Project-Only Memory

**Status:** Live on web and Windows. iOS/Android rollout announced but timing uncertain.

**How it works:** Project-only memory isolates context to conversations within a single project. ChatGPT creates automatic memory snippets from project chats but cannot access full conversation history. Outside saved memories are excluded.

**Critical constraints:**
- Must be set at project creation — cannot be retrofitted to existing projects.
- Requires three memory toggles enabled: Memory, Reference Saved Memories, and Reference Chat History (Settings > Personalization > Memory).
- No browsable memory list — you cannot see what the project has remembered or edit individual project memories.
- **Platform gap:** Project-only memory is currently web + Windows only. If sessions run on iOS voice mode, the project-only memory isolation may not function until the mobile rollout completes.

**Session accumulation:** No evidence of hard chat-count limits per project, but since memory is opaque (no list to browse/edit), stale context from old sessions could pollute future sessions. The spec's approach of one chat per session is the correct mitigation — but there's no way to prune what the project "remembers" short of deleting chats.

**Source:** [Yahoo Tech](https://tech.yahoo.com/ai/articles/now-limit-chatgpt-memory-single-132811095.html), [Unite.AI](https://www.unite.ai/how-to-use-chatgpts-project-memory/), [TechRadar](https://www.techradar.com/ai-platforms-assistants/chatgpt/chatgpt-project-only-memory-is-live-and-it-might-change-how-you-work-with-ai)

### 2. Voice Mode + Image Upload on iOS

**Status:** Live for Plus/Pro subscribers on iOS and Android.

**Capabilities during voice:**
- Take a photo (opens camera)
- Upload a photo from library
- Share screen (broadcasts live screen)
- Video camera input
- After voice ends, transcript is added to the same text chat

**Constraints:**
- Daily usage limits based on subscription tier (not publicly quantified per plan).
- Screen share and video are separate from photo upload — accessed via three-dot menu.
- No file upload (PDF, etc.) during voice — only images and screen share.
- Image handling in voice is visual-only; the model sees the image but OCR accuracy on handwritten math or small printed text is not guaranteed.

**Implication for the spec:** The image protocol (full page + close-up crop + pasted text fallback) is well-designed given these constraints. The pasted-text fallback is critical because voice mode image OCR on worksheets with small numbers will be unreliable.

**Source:** [LearnPrompting](https://learnprompting.org/blog/how-to-use-openai-chatgpt-advanced-voice-mode), [Neowin](https://www.neowin.net/news/chatgpt-mobile-apps-now-support-video-and-screensharing-in-advanced-voice-mode/), [9to5Mac](https://9to5mac.com/2024/12/12/chatgpt-advanced-voice-mode-adding-video-and-screen-sharing-input-plus-a-santa-mode/)

### 3. Study Mode Limitations for Math

**Status:** Available but problematic for this use case.

**Key findings:**
- ChatGPT provided incorrect step-by-step solution approaches 42% of the time (UPenn study, ~1000 students). Arithmetic errors were only 8%, but the reasoning/method was wrong nearly half the time.
- Study Mode is not grade-calibrated. OpenAI designed it for advanced learners — "not tuned for the language, pacing, or vocabulary that elementary and middle school students need."
- Bypass is trivial — students can request full solutions, and Study Mode sometimes provides direct answers despite being designed not to.
- Students who used AI tutoring solved more practice problems but performed 17% worse on actual tests (UPenn). AI-tutored students showed false confidence — believed they learned more than they did.

**Implication for the spec:** The spec's decision to keep Study Mode off by default is correct. The custom project instructions (hint ladder, one-step-at-a-time, no-answer-reveal) are more reliable than Study Mode for controlling behavior. Study Mode adds unpredictability without adding value for a grade-4 child.

**Source:** [Hechinger Report / UPenn Study](https://hechingerreport.org/kids-chatgpt-worse-on-tests/), [StarSpark comparison](https://www.starspark.ai/blog/starspark-vs.-chatgpt-study-mode-the-best-ai-math-tutor-for-k-12), [OpenAI Study Mode announcement](https://openai.com/index/chatgpt-study-mode/)

### 4. Privacy and Children Under 13

**This is the most material finding.**

- **OpenAI's terms state the service is not directed to children under 13, and they do not knowingly collect personal data from children under 13.**
- Parental controls exist but are designed for teens (13+), not younger children.
- When "Improve the model for everyone" is enabled, conversations (including voice transcripts, uploaded images, and model outputs) may be used for training.
- Users can opt out via Settings > Data Controls or OpenAI's privacy portal ("do not train on my content").
- Audio/video clips from voice chats are stored alongside transcription and deleted within 30 days of chat deletion.

**Implication for the spec:** The child in this scenario is 9 years old — below OpenAI's stated minimum age of 13. This doesn't prevent a parent from running the session themselves (the parent is the account holder and primary operator), but the child is directly interacting with the service via voice. The spec's privacy branch should be strengthened to acknowledge this age-policy gap explicitly. The parent should absolutely disable "Improve the model for everyone" and understand that the child's voice data is being processed even with training opted out.

**Source:** [OpenAI Parental Controls FAQ](https://help.openai.com/en/articles/12315553-parental-controls-on-chatgpt-faq), [OpenAI Data Controls](https://help.openai.com/en/articles/7730893-data-controls-faq), [OpenAI Privacy Policy](https://openai.com/policies/privacy-policy/)

## Trade-offs

| Decision | Upside | Downside |
|----------|--------|----------|
| Custom project instructions over Study Mode | Full control over tutor behavior, hint ladder, praise calibration | Requires more parent involvement to prompt correctly; no adaptive difficulty |
| Project-only memory | Session isolation, no cross-project contamination | Opaque memory (can't see/edit what it remembers), iOS support uncertain |
| Voice mode for tutoring | Natural interaction for a 9-year-old, hands-free | Daily usage limits, OCR unreliable on small text, 42% incorrect step-by-step reasoning |
| One chat per session | Clean session logs, prevents context pollution | Can't reference prior sessions easily if child revisits a topic |
| Parent as operator (child as user) | Works around under-13 age restriction | Still a gray area; child's voice/image data is processed |

## Recommendations

1. **Disable "Improve the model for everyone"** before the first session. This is non-negotiable given the child's age.

2. **Drop Study Mode from the spec entirely.** It adds unpredictability, isn't grade-calibrated, and the custom project instructions already provide better pedagogical control. Remove it as a branch point — it's a liability, not an option.

3. **Add explicit accuracy warnings to the spec.** The 42% incorrect-approach rate means the parent must actively verify the tutor's mathematical guidance. The spec should include a parent-facing note: "ChatGPT gets the method wrong ~40% of the time on math. You are the accuracy backstop."

4. **Strengthen the pasted-text protocol.** Given OCR limitations in voice mode, the spec should elevate "paste exact problem text" from a fallback to a default for any problem with numbers, units, or multiple-choice options.

5. **Acknowledge the under-13 age policy gap** in the privacy section. The parent should understand they're operating outside OpenAI's stated terms for the child's direct interaction, even though the parent owns the account.

6. **Verify project-only memory works on iOS** before relying on it. If it's web-only at session time, the memory isolation guarantee breaks. The parent could start the chat on web (where project-only works), then switch to voice on iOS.

7. **Consider periodic chat pruning.** Since project memory is opaque and can accumulate stale context, recommend deleting old session chats monthly or when the tutor starts referencing outdated context.

## Open Questions (Remaining)

- Does project-only memory function correctly when a chat is started on web but voice is used on iOS mobile? (Cross-platform memory sync is undocumented.)
- What are the actual daily voice mode usage limits for Plus vs Pro subscribers? (Not publicly documented per-plan.)
- Has the parent already created a project with default memory that needs to be recreated?
- What worksheet curriculum is being used? (Affects whether skill-identification accuracy matters.)
- Would the parent prefer to always paste problem text rather than rely on image OCR?
