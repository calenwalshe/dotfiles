# Clarify Brief

- **Slug:** tutor-upload
- **Timestamp:** 20260403T223004Z
- **Status:** draft

## Idea (verbatim)

Build tutorupload.calenwalshe.com — a lightweight web app where the parent uploads cropped worksheet images, which are grouped into a session pack with a shareable link. The parent opens that link on their laptop during a ChatGPT tutoring session and pastes the crops into the chat. Runs on the existing Caddy + Docker infrastructure on this server.

## Goal

A working web app at tutorupload.calenwalshe.com where the parent can upload cropped worksheet images from any device, get a session pack link, and open that link on the laptop to quickly copy/paste each crop into a ChatGPT tutoring chat — eliminating the device-transfer friction in the worksheet prep protocol.

## Non-Goals

- ChatGPT API integration or any direct connection to OpenAI
- OCR, text extraction, or any image processing on the server side
- User accounts, authentication, or multi-user support
- Long-term image storage or archival — session packs are ephemeral
- Mobile app — this is a web-only tool
- Curriculum management, progress tracking, or tutoring logic
- Replacing the ChatGPT Project or session workflow — this is a prep tool only

## Constraints

- Must run on the existing server (144.202.81.218) using Caddy reverse proxy + Docker
- Cloudflare DNS is managed via CF_API_TOKEN / CF_ZONE_ID already in agent-stack/.env
- TLS terminated by Cloudflare (proxied) with self-signed certs on Caddy origin
- Must follow the existing Caddyfile pattern (subdomain block + reverse_proxy to container)
- Lightweight — no heavy frameworks, databases optional (filesystem or SQLite at most)
- Images must be easy to copy/paste or drag into ChatGPT from the session pack page
- Parent is the sole user — no need for auth, rate limiting, or abuse prevention at this stage

## Assumptions

- Cloudflare DNS for calenwalshe.com is accessible via the CF_API_TOKEN in agent-stack/.env
- The existing Caddy container can be reloaded to pick up new Caddyfile entries
- Docker Compose in agent-stack/ is the deployment mechanism for new services
- The parent uploads crops from a phone (camera roll) or laptop (file system)
- Session packs are short-lived — auto-expire after 24-48 hours is acceptable
- Image sizes are small (worksheet crops, typically <1MB each)
- The session pack page needs to make it trivial to get images into ChatGPT (click-to-copy, drag-and-drop, or direct download)

## Open Questions

- What's the fastest image-to-ChatGPT transfer method from a browser? (Click-to-copy-image, drag-and-drop from browser to ChatGPT, download-then-upload?)
- Should the upload interface support camera capture directly (for phone-based cropping) or just file upload?
- What's the right expiry window for session packs — 24 hours, 48 hours, or manual delete?
- Should the service use the existing Garage S3-compatible storage for images, or just local filesystem in the container?
- Does the docker-compose in agent-stack need a new service entry, or should this be a standalone container?
