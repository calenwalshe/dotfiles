Structure (implement in this order):

**1. Imports and constants:**

import asyncio, aiohttp, base64, io, json, os, sys, uuid, ipaddress

Constants:

- CF_BYPASS_URL = os.environ.get("CF_BYPASS_URL", "http://cf-bypass-worker:9160")
- SMART_ROUTER_URL = os.environ.get("SMART_ROUTER_URL", "http://smart-router:9080")
- ARTIFACT_DIR = Path(os.environ.get("ARTIFACT_DIR", "/var/lib/openclaw/workspace/screenshot-artifacts"))
- CF_INDICATORS = ["just a moment", "attention required", "cloudflare", "cdn-cgi", "verify you are human", "please wait while your request is being verified", "captcha", "access denied"]
- BLOCKED_HOSTS = {"169.254.169.254", "metadata.google.internal", "metadata.internal"}
- _bypass_semaphore = asyncio.Semaphore(1)  # module-level, shared across all coroutines

**2. validate_url(url) -> tuple[bool, str]:**
Use ipaddress + urlparse. Block schemes other than http/https. Block BLOCKED_HOSTS. Block private/loopback/link-local IPs. Return (True, "") on pass, (False, reason) on failure. Exact implementation from RESEARCH.md Pattern 5.

**3. _classify_screenshot(blocker, page_url, title, screenshot_b64, nav_latency_ms) -> tuple[str, float]:**
Implement exactly as in RESEARCH.md Pattern 2. Rule-based fast path:
- CF_INDICATORS in combined string → BLOCKED_CHALLENGE (0.95 if blocked flag set, 0.90 otherwise)
- screenshot_b64 missing or len < 500 → BLANK_PAGE (0.90)
- nav_latency_ms < 200 → BLANK_PAGE (0.70)
- blocked but no CF signal → SOFT_BLOCK (0.75)
- title empty or "untitled" or "about:blank" → DEGRADED_CONTENT (0.65)
- Otherwise → PASS_CONTENT (0.85)
Confidence < 0.70 is the threshold for LLM fallback (returned as-is from fast path; caller decides whether to invoke LLM).

**4. _classify_with_llm(screenshot_b64, url) -> tuple[str, float]:**
Call smart-router:9080 with model "gemini-vision". Construct an OpenAI-compatible request with a base64 JPEG image in the messages. Prompt: "Classify this screenshot into exactly one of: PASS_CONTENT, BLOCKED_CHALLENGE, BLANK_PAGE, DEGRADED_CONTENT, SOFT_BLOCK. Also provide a confidence score 0.0-1.0. Reply in JSON format: {\"class\": \"...\", \"confidence\": 0.0}. PASS_CONTENT means normal website content. BLOCKED_CHALLENGE means a Cloudflare or CAPTCHA challenge page. BLANK_PAGE means empty or nearly empty. DEGRADED_CONTENT means content present but sparse or broken. SOFT_BLOCK means access denied without a challenge page." Parse response JSON. On any error fall back to ("SOFT_BLOCK", 0.50).

**5. capture_screenshot(url, timeout_s=30) -> dict:**
Implement the browser session lifecycle from RESEARCH.md Pattern 1. Session name: f"l1-{uuid4().hex[:12]}". Always cleanup in finally block. Return nav_json on success, {"ok": False, "error": ...} on failure.

**6. attempt_bypass(url, timeout_s=60) -> dict:**
Implement with _bypass_semaphore as in RESEARCH.md Pattern 3. If semaphore wait would exceed 90s (use asyncio.wait_for), return {"ok": False, "error": "bypass worker busy — try again shortly"}.

**7. resize_and_save(screenshot_b64, filename_stem) -> str:**
Decode base64 PNG. Open with PIL. Resize to max 1280px wide (preserve aspect ratio, use Image.LANCZOS). Save as JPEG quality=80 to ARTIFACT_DIR/{YYYY-MM-DD}/{filename_stem}.jpg. Create dirs as needed. Return absolute path string.

**8. build_caption(url, cls, conf, latency_ms, challenge_status="") -> str:**
CLASS_LABELS = {"PASS_CONTENT": "OK", "BLOCKED_CHALLENGE": "Blocked", "BLANK_PAGE": "Blank", "DEGRADED_CONTENT": "Partial", "SOFT_BLOCK": "Soft block", "NAV_FAILURE": "Failed"}
Format: URL (truncated to 80 chars), Classification label + confidence as integer %, Load time in seconds, Challenge line if present. Cap total at 900 chars.

**9. save_metadata(url, cls, conf, latency_ms, screenshot_path) -> None:**
Write a line of JSON to ARTIFACT_DIR/metadata.jsonl. Fields: url, classification, confidence, latency_ms, screenshot_path, timestamp (ISO 8601 UTC). Create file if missing, append mode.

**10. async main(url, chat_id=None) -> dict:**
Full pipeline:
- validate_url → on fail return {ok: False, error, class: "NAV_FAILURE", confidence: 1.0}
- capture_screenshot(url, timeout_s=30) → on fail return error dict
- _classify_screenshot on result
- If class == "BLANK_PAGE": asyncio.sleep(2.0) → retry capture_screenshot(url, timeout_s=45) → re-classify
- If class == "BLOCKED_CHALLENGE":
  - Save challenge screenshot (resize_and_save with stem "challenge-{uuid4().hex[:8]}")
  - Return intermediate result with key "challenge_screenshot_path" set (for agent to show user before bypass)
  - Then call attempt_bypass(url, timeout_s=60)
  - On bypass success: re-classify bypass screenshot; if still BLOCKED, mark bypassed=False
  - On bypass failure: use challenge screenshot as final; cls stays BLOCKED_CHALLENGE
- If confidence < 0.70 and class not in {"BLOCKED_CHALLENGE", "NAV_FAILURE"}: call _classify_with_llm
- resize_and_save final screenshot
- save_metadata
- build_caption
- Return: {ok: True, class: cls, confidence: conf, screenshot_path: str, caption: str, bypassed: bool (if CF involved), challenge_screenshot_path: str (if CF involved)}

**11. Entry point:**
```python
if __name__ == "__main__":
    if len(sys.argv) < 2:
        print(json.dumps({"ok": False, "error": "Usage: screenshot_tool.py <url> [chat_id]"}))
        sys.exit(0)

    chat_id = sys.argv[2] if len(sys.argv) > 2 else None
    result = asyncio.run(main(url, chat_id))
    print(json.dumps(result))
```
